eqko

eqko is my audiovisual performance project which stemmed from the principle of video feedback.


Take a look : )


Eqko showreel 2k17 from eqko on Vimeo.



The technique is inspired by analog video feedback, in which a camera and a cathode-ray tube are connected in a closed loop. Others, such as algorithm generation of geometric figures and glitch-art are incorporated in the process to enhance the chaotic and hypnotic rendering.

  • Website Project : www.eqko.net
  • Date : January 2013, and will continue to loop.
  • Tags : digital feedback ∞ loud noises ∞ glitch art ∞ algorithm generation ∞ chaos hypnosis

DIY Geiger-Counter

Interested for a long time in the subject of radioactivity and and its related historical facts, I decided (just after Fukushima's catastrophe) to build my own DIY Geiger Counter in late 2011.

Pretty nice project based on Arduino with lots of solderings and fragiles components, doing it was also about getting concrete informations about radiations and how they work.
I also needed to ship the Geiger Müller Tube from a private and obscure online shop in China so it would work properly, which was kind of fun and scary, but it worked, and of course radiations were in the average!

  • Website Project : DIYGeigerCounter - GK-B5
  • Date : August 2011
  • Tags : Geiger Counter, Radioactivity, Arduino, Electronics

Arduino for the LoL Shield

The LoL Shield is a charlieplexed LED matrix for the Arduino. The LEDs are individually addressable, so you can use it to display anything in a 9×14 grid (126 led of two pins = 252 holes to solder).
Scroll text, play games, display images, or anything else you want to do :)

  • Website Project : LoL Shield
  • Date : February 2015
  • Tags : Lots of LED, Lots of soldering, Arduino, DIY

Multitier software architecture : Noise Monitoring

This is my work with Arkamys, a service company in audio signal processing and research in innovative software.

I designed a software prototype to recover an data from an earphone's device with an Android App. My part was to create the app that record external/ambient noises every 5 seconds, then save all the data, the noises location and their type in a database.

At the end of day/week/month, the user can view the noises data on a map and on a chart to dynamically see which places are noisier and which type of noises are dangerous for ear's users.

The architecture was also designed so a guest could view all community noises on the map.

Here is the list of the different technologies & concepts I used to do this project :


Android • SQL • Javascript • REST • Leaflet • Chart.JS • jQuery• PHP • OpenStreetMap • HTML/CSS • Twitter Bootstrap • Object-Relational Mapping (ORM) • Model-View Controller (MVC)
